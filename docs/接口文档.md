# API 接口文档 — LLM Wiki Approach

## 设计理念

采用"万物皆MD"架构：API 返回 JSON 信封，核心内容为 Markdown 字符串。知识图谱从 vault 中的 `[[wikilinks]]` 自动派生，无需独立存储图结构。

## 基础信息

- Base URL: `http://localhost:8000`
- 响应格式: JSON
- 认证: 无（MVP 阶段）

### 文档管理

#### 上传并解析文档

```
POST /api/documents/upload
Content-Type: multipart/form-data
```

参数: `file` (PDF/MD/TXT，最大 512MB)

响应:
```json
{
  "document_id": "doc_a1b2c3d4",
  "status": "parsed",
  "chapter_count": 12,
  "total_chars": 385000,
  "vault_paths": ["textbooks/病理学"]
}
```

副作用: 解析后的章节同时写入 `data/vault/textbooks/{title}/` 目录。

#### 上传并返回完整解析结果

```
POST /api/parse/upload
Content-Type: multipart/form-data
```

返回完整的 ParsedDocument JSON（用于前端章节预览）。

#### 列出已上传文档

```
GET /api/documents
```

响应:
```json
{
  "documents": [
    {
      "document_id": "doc_a1b2c3d4",

}
```

### 知识提取

#### 触发 LLM 知识点提取

```
POST /api/extraction/run
Content-Type: application/json
```

请求:
```json
{"document_id": "doc_a1b2c3d4"}
```

响应:
```json
{
  "document_id": "doc_a1b2c3d4",
  "status": "ready",
  "concepts_written": 42,
  "vault_paths": ["concepts/炎症.md", "concepts/细胞损伤.md"]
}
```

副作用: 每个知识点生成一个 active Obsidian-flavored MD 文件到 `data/vault/concepts/`。同名 active concept 不覆盖；后续同名写入带 `__{textbook_id}__{chapter_id}__{n}` 后缀的文件，并创建/更新 merge candidate。

### Merge 决策与执行

#### 手动扫描同名候选

```
POST /api/merge/scan
```

响应:
```json
{
  "decisions": [
    {
      "decision_id": "merge_same_name_细胞膜",
      "status": "candidate",
      "trigger": "same_name",
      "method": "deterministic_same_name",
      "affected_nodes": ["concepts/细胞膜.md", "concepts/细胞膜__doc_x__ch_y__1.md"],
      "result_name": "细胞膜",
      "result_node": "concepts/细胞膜.md",
      "reason_summary": "Same canonical_name: 细胞膜"
    }
  ]
}
```

#### 外部 scanner 写入 candidate

```
POST /api/merge/decisions
Content-Type: application/json
```

请求:
```json
{
  "affected_nodes": ["concepts/A.md", "concepts/B.md"],
  "result_name": "A",
  "reason_summary": "A and B describe the same concept",
  "reasoning_md": "## Reasoning\n\n..."
}
```

#### 列出决策

```
GET /api/merge/decisions?status=candidate
```

#### 读取决策详情

```
GET /api/merge/decisions/{decision_id}
```

响应:
```json
{
  "path": "decisions/merge/merge_same_name_细胞膜.md",
  "frontmatter": {"decision_id": "merge_same_name_细胞膜", "status": "candidate"},
  "content_md": "# Merge decision: 细胞膜\n...",
  "wikilinks": ["细胞膜", "细胞膜__doc_x__ch_y__1"]
}
```

#### 执行 merge

```
POST /api/merge/execute
Content-Type: application/json
```

请求:
```json
{
  "decision_id": "merge_same_name_细胞膜",
  "affected_nodes": ["concepts/细胞膜.md", "concepts/细胞膜__doc_x__ch_y__1.md"],
  "result_name": "细胞膜",
  "frontmatter": {"category": "核心概念", "aliases": []},
  "body": "# 细胞膜\n\n综合定义...\n"
}
```

成功副作用:

- 写入 `concepts/{safe_result_name}.md`
- 旧节点移动到 `archive/concepts/{decision_id}/`
- active vault 内容中的旧 wikilink 重写到新节点；alias label 保留
- decision 更新为 `applied`

错误:

- `422`: payload 缺字段或格式不合法；不移动文件
- `409`: result path 已存在且未纳入 `affected_nodes`

## Vault 文件格式 (Obsidian-compatible)

每个 active 概念文件位于 `data/vault/concepts/`。首个同名概念使用 `{safe_name}.md`；后续同名概念使用 `{safe_name}__{textbook_id}__{chapter_id}__{n}.md`。完整规范以 `AGENTS.md` 的 `SSOT: Concept Vault Schema` 为准：

```markdown
---
id: concept_炎症
canonical_name: 炎症
status: active
category: 核心概念
aliases: [inflammation, 炎性反应]
merge_decisions: []
sources:
  - textbook_id: book_01
    chapter_id: ch_004
    page_start: 78
    page_end: 82
    definition: 炎症是具有血管系统的活体组织对各种损伤因子的刺激所发生的防御性反应。
    evidence: 炎症(inflammation)是具有血管系统的活体组织对各种损伤因子...
---

# 炎症

炎症是具有血管系统的活体组织对各种损伤因子的刺激所发生的防御性反应。

## 关系

- 前置依赖: [[细胞损伤]]
- 包含: [[急性炎症]], [[慢性炎症]]
- 并列: [[免疫应答]]

## 来源证据

> 炎症(inflammation)是具有血管系统的活体组织对各种损伤因子...

来源: 病理学 ch_004
```

关系类型:
| 类型 | 含义 |
|------|------|
| prerequisite | 前置依赖 |
| contains | 包含 |
| parallel | 并列 |
| applies_to | 应用于 |

## 错误码

| 状态码 | 含义 |
|--------|------|
| 400 | 请求参数错误 |
| 409 | merge result path 冲突，需把冲突节点纳入 affected_nodes |
| 404 | 资源不存在 |
| 413 | 文件过大 |
| 422 | 不支持的文件格式；或 merge execute payload 不合法 |
| 502 | LLM 服务返回错误 |
| 503 | LLM 配置缺失 |
