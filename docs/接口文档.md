# 接口文档

本文档是后端 API 与跨端 JSON schema 的人工版引用入口。修改后端 endpoint、请求/响应 JSON、错误码或核心 schema 时，必须同步更新本文档。

运行后端后，FastAPI 自动文档可访问：

- Swagger UI: `/docs`
- OpenAPI JSON: `/openapi.json`

## 1. API 状态约定

| 状态 | 含义 |
| --- | --- |
| implemented | 当前代码已实现，可直接调用。 |
| contract | 前后端并行开发契约，后端下一步按此实现；前端可据此对接 mock。 |
| reserved | 为后续整合/RAG/对话预留路径，当前不展开实现。 |

当前前端知识图谱交互层以 `implemented` API 为准；不得把 `reserved` API 当作可用功能。

## 2. 稳定性约定

| 对象 | 状态 | 说明 |
| --- | --- | --- |
| `ParsedDocument` | MVP stable | 解析结果结构。字段可增，不应破坏现有字段。 |
| `KnowledgeGraph` | implemented | 图谱展示与节点详情的核心结构；节点和边包含 `source_ref`。 |
| `GraphView` | implemented | 前端画图专用结构，包含节点大小、颜色键、legend、频次。 |
| `SourceRef` | implemented | 节点/边跳转原文与高亮 evidence 的定位结构。 |
| RAG schemas | reserved | RAG endpoint 暂不属于当前知识图谱交互层。 |

## 3. Implemented API

### `GET /api/health`

健康检查。

Response `200`:

```json
{
  "status": "ok"
}
```

### `POST /api/parse/upload`

当前已实现的单文件同步解析接口。保留给 parse preview；内部同样会注册 document，返回值仍保持旧版 `ParsedDocument`。

Request:

- Content-Type: `multipart/form-data`
- Field: `file`
- Supported suffixes: `.pdf`, `.md`, `.markdown`, `.txt`
- Max size: `512 MB`

Response `200`: `ParsedDocument`

Error responses:

| Status | 条件 | Body |
| --- | --- | --- |
| `400` | 缺少文件名 | `{"detail": "No filename provided."}` |
| `413` | 文件超过 512 MB | `{"detail": "File too large (max 512 MB)."}` |
| `422` | 不支持的文件类型 | `{"detail": "Unsupported file type: ..."} ` |
| `500` | parser 运行失败 | `{"detail": "Parse error: ..."}` |

## 4. Graph Interaction Implemented API

本节覆盖赛题“上传解析 → 单书图谱生成 → 图谱交互展示”所需 API。当前实现为同步返回；大 PDF 或多文档耗时变长后，可沿用同一响应结构返回 `job_id + running`。

### 4.1 Documents

#### `POST /api/documents/upload` implemented

上传单个文档，解析并注册为可查询 document。

Request:

- Content-Type: `multipart/form-data`
- Field: `file`

Response `200`:

```json
{
  "document_id": "doc_ab12cd34",
  "status": "parsed",
  "document": {
    "textbook_id": "doc_ab12cd34",
    "filename": "lab2-1.md",
    "title": "lab2-1",
    "total_pages": null,
    "total_chars": 20383,
    "chapters": [],
    "format": "markdown",
    "source_path": null
  }
}
```

Async-compatible response:

```json
{
  "document_id": "doc_ab12cd34",
  "job_id": "job_001",
  "status": "parsing",
  "document": null
}
```

当前实现总是同步解析，成功返回 `200`；错误码同 `POST /api/parse/upload`。

#### `GET /api/documents` implemented

列出已上传/解析文档，用于左侧教材管理区。

Response `200`:

```json
{
  "documents": [
    {
      "document_id": "doc_ab12cd34",
      "filename": "lab2-1.md",
      "title": "lab2-1",
      "format": "markdown",
      "size_bytes": 20383,
      "status": "parsed",
      "chapter_count": 6,
      "total_chars": 20383,
      "graph_status": "ready"
    }
  ]
}
```

#### `GET /api/documents/{document_id}` implemented

读取完整 `ParsedDocument`。

Response `200`:

```json
{
  "document": {
    "...": "ParsedDocument"
  }
}
```

Error `404`: document 不存在。

#### `GET /api/documents/{document_id}/chapters` implemented

读取章节摘要，不返回大段正文。

Response `200`:

```json
{
  "document_id": "doc_ab12cd34",
  "chapters": [
    {
      "chapter_id": "ch_003",
      "title": "背景知识",
      "char_count": 11000,
      "page_start": null,
      "page_end": null,
      "line_start": 16,
      "line_end": 353
    }
  ]
}
```

Error `404`: document 不存在。

#### `GET /api/documents/{document_id}/chapters/{chapter_id}` implemented

读取单章正文，供右侧原文面板或 source 高亮使用。

Response `200`:

```json
{
  "document_id": "doc_ab12cd34",
  "chapter": {
    "...": "ParsedChapter"
  }
}
```

Error `404`: document 或 chapter 不存在。

### 4.2 Graph Extraction

#### `POST /api/graphs/extract` implemented

为一个已解析文档生成单书知识图谱。MVP 可同步调用 LLM；后续可返回 job。

Request:

```json
{
  "document_id": "doc_ab12cd34",
  "force": false
}
```

Response `200` when ready:

```json
{
  "graph_id": "graph_doc_ab12cd34",
  "document_id": "doc_ab12cd34",
  "status": "ready",
  "graph": {
    "...": "KnowledgeGraph"
  }
}
```

当前实现会创建 job 记录，但抽取过程同步完成；`force=false` 且图谱已存在时直接返回已有图谱。

Response `202` when running:

```json
{
  "job_id": "job_001",
  "graph_id": null,
  "document_id": "doc_ab12cd34",
  "status": "running"
}
```

Error responses:

| Status | 条件 |
| --- | --- |
| `400` | 缺少 `document_id` |
| `404` | document 不存在 |
| `503` | LLM 配置缺失 |
| `502` | LLM provider/response 错误 |
| `500` | 其他抽取异常 |

#### `GET /api/documents/{document_id}/graph` implemented

读取某文档当前图谱。

Response `200`:

```json
{
  "graph_id": "graph_doc_ab12cd34",
  "document_id": "doc_ab12cd34",
  "status": "ready",
  "graph": {
    "...": "KnowledgeGraph"
  }
}
```

Error `404`: document 或 graph 不存在。

#### `GET /api/graphs/{graph_id}` implemented

按 graph ID 读取图谱。

Response `200`: 同 `GET /api/documents/{document_id}/graph`。

Error `404`: graph 不存在。

### 4.3 Graph View

#### `POST /api/graphs/view` implemented

返回前端可直接渲染的 graph view。单文档时用于基本图谱；多文档时用于来源区分和频次可视化。

Request:

```json
{
  "document_ids": ["doc_a", "doc_b"],
  "mode": "per_document"
}
```

Allowed `mode`:

- `per_document`: 不做语义合并，只按文档来源分色。
- `frequency_preview`: 用 name/alias 归一化做轻量频次统计；不等同正式跨教材整合。

Response `200`: `GraphView`

```json
{
  "view_id": "view_001",
  "mode": "frequency_preview",
  "nodes": [
    {
      "id": "doc_a_node_001",
      "label": "行波进位加法器",
      "category": "方法",
      "document_id": "doc_a",
      "chapter_id": "ch_003",
      "source_count": 2,
      "source_documents": ["doc_a", "doc_b"],
      "size": 22,
      "color_key": "doc_a"
    }
  ],
  "edges": [
    {
      "id": "edge_001",
      "source": "doc_a_node_001",
      "target": "doc_a_node_002",
      "relation_type": "prerequisite",
      "label": "前置依赖",
      "color_key": "prerequisite"
    }
  ],
  "legend": {
    "documents": [
      {
        "document_id": "doc_a",
        "title": "lab2-1",
        "color_key": "doc_a"
      }
    ],
    "relations": [
      {
        "relation_type": "prerequisite",
        "label": "前置依赖",
        "color_key": "prerequisite"
      }
    ]
  }
}
```

Error responses:

| Status | 条件 |
| --- | --- |
| `400` | `document_ids` 为空或 `mode` 非法 |
| `404` | document 或对应 graph 不存在 |

### 4.4 Detail, Source, Search

#### `GET /api/graphs/{graph_id}/nodes/{node_id}` implemented

节点点击详情。

Response `200`:

```json
{
  "node": {
    "...": "KnowledgeNode"
  },
  "chapter": {
    "document_id": "doc_a",
    "chapter_id": "ch_003",
    "title": "背景知识"
  }
}
```

Error `404`: graph、node 或对应 document 不存在。

#### `GET /api/graphs/{graph_id}/edges/{edge_id}` implemented

边点击详情。

Response `200`:

```json
{
  "edge": {
    "...": "KnowledgeEdge"
  },
  "source_node": {
    "id": "doc_a_node_001",
    "name": "全加器"
  },
  "target_node": {
    "id": "doc_a_node_002",
    "name": "行波进位加法器"
  }
}
```

Error `404`: graph 或 edge 不存在。

#### `GET /api/source/{source_id}` implemented

返回可高亮的章节正文。前端按 `highlight.char_start/char_end` 高亮 evidence。

Response `200`:

```json
{
  "source_id": "doc_a:ch_003:ev_001",
  "document_id": "doc_a",
  "chapter_id": "ch_003",
  "chapter_title": "背景知识",
  "content": "完整 cleaned chapter content...",
  "highlight": {
    "char_start": 1280,
    "char_end": 1325
  }
}
```

Error `404`: source、document 或 chapter 不存在。

#### `GET /api/graphs/{graph_id}/search?q=加法器` implemented

搜索节点。MVP 使用 `name/aliases/definition` 子串匹配；后续可替换为 embedding 检索。

Response `200`:

```json
{
  "matches": [
    {
      "node_id": "doc_a_node_001",
      "name": "行波进位加法器",
      "match_field": "name",
      "score": 1.0
    }
  ]
}
```

Error responses:

| Status | 条件 |
| --- | --- |
| `400` | `q` 为空 |
| `404` | graph 不存在 |

### 4.5 Jobs

#### `GET /api/jobs/{job_id}` implemented

查询解析或图谱抽取任务状态。

Response `200`:

```json
{
  "job_id": "job_001",
  "kind": "graph_extract",
  "status": "running",
  "progress": 0.45,
  "message": "Extracting chapter 3/6",
  "result": null,
  "error": null
}
```

Error `404`: job 不存在。

Allowed `status`:

- `queued`
- `running`
- `parsed`
- `ready`
- `failed`

## 5. JSON Schemas

### `ParsedDocument`

```ts
type ParsedDocument = {
  textbook_id: string;
  filename: string;
  title: string;
  total_pages: number | null;
  total_chars: number;
  chapters: ParsedChapter[];
  format: "pdf" | "markdown" | string;
  source_path: string | null;
};
```

### `ParsedChapter`

`content` 是 parser 清洗后的正文，不保证与原始文件字节完全一致。Markdown 的 `line_start/line_end` 指向原文件行号；PDF 通常只有页码。

```ts
type ParsedChapter = {
  chapter_id: string;
  title: string;
  content: string;
  char_count: number;
  page_start: number | null;
  page_end: number | null;
  level: number | null;
  line_start: number | null;
  line_end: number | null;
  assets: AssetReference[];
};
```

### `AssetReference`

```ts
type AssetReference = {
  kind: string;
  label: string;
  path: string;
  line: number | null;
};
```

### `KnowledgeGraph`

```ts
type KnowledgeGraph = {
  graph_id: string | null;
  document_id: string;
  source_document: {
    filename?: string;
    title?: string;
    format?: string;
    source_path?: string | null;
  };
  nodes: KnowledgeNode[];
  edges: KnowledgeEdge[];
  stats: {
    node_count: number;
    edge_count: number;
  };
};
```

### `KnowledgeNode`

`evidence` 必须尽量是章节原文中的连续短句。后端生成 `source_ref` 时会优先 exact match，失败则降级。当前抽取器生成的节点固定返回 `source_ref`。

```ts
type KnowledgeNode = {
  id: string;
  name: string;
  aliases: string[];
  definition: string;
  category: string;
  textbook_id: string;
  chapter_id: string;
  page_start: number | null;
  evidence: string;
  confidence: number;
  source_ref: SourceRef;
};
```

### `KnowledgeEdge`

Allowed `relation_type`:

- `prerequisite`
- `contains`
- `parallel`
- `applies_to`

```ts
type KnowledgeEdge = {
  id: string;
  source: string;
  target: string;
  relation_type: "prerequisite" | "contains" | "parallel" | "applies_to";
  description: string;
  evidence: string;
  confidence: number;
  source_ref: SourceRef;
};
```

### `SourceRef`

`char_start/char_end` 指向 `ParsedChapter.content` 的 cleaned content。Markdown 行号由 chapter 行号近似推导；PDF 行号通常为 `null`。

```ts
type SourceRef = {
  document_id: string;
  chapter_id: string;
  source_id: string;
  page_start: number | null;
  page_end: number | null;
  line_start: number | null;
  line_end: number | null;
  char_start: number | null;
  char_end: number | null;
  match_method:
    | "exact"
    | "whitespace_normalized"
    | "punctuation_normalized"
    | "chapter_fallback";
};
```

### `GraphView`

```ts
type GraphView = {
  view_id: string;
  mode: "per_document" | "frequency_preview";
  nodes: GraphViewNode[];
  edges: GraphViewEdge[];
  legend: GraphLegend;
};

type GraphViewNode = {
  id: string;
  label: string;
  category: string;
  document_id: string;
  chapter_id: string;
  source_count: number;
  source_documents: string[];
  size: number;
  color_key: string;
};

type GraphViewEdge = {
  id: string;
  source: string;
  target: string;
  relation_type: "prerequisite" | "contains" | "parallel" | "applies_to";
  label: string;
  color_key: string;
};

type GraphLegend = {
  documents: Array<{
    document_id: string;
    title: string;
    color_key: string;
  }>;
  relations: Array<{
    relation_type: string;
    label: string;
    color_key: string;
  }>;
};
```

## 6. Reserved API

以下路径为后续跨教材整合、RAG、教师反馈保留；当前知识图谱交互层不依赖它们。

| API | 状态 | 用途 |
| --- | --- | --- |
| `POST /api/integration/merge` | reserved | 跨教材语义对齐、merge/keep/remove 决策。 |
| `GET /api/integration/{integration_id}` | reserved | 读取整合图谱和压缩统计。 |
| `POST /api/rag/index` | reserved | 建立 RAG 索引。 |
| `POST /api/rag/query` | reserved | 带引用问答。 |
| `GET /api/rag/status` | reserved | 查询索引状态。 |
| `POST /api/chat` | reserved | 教师反馈与决策修改。 |
