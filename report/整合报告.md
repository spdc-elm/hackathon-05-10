## 2. 整合决策摘要

当前系统把 concept 抽取与 merge 执行分开：

- 抽取阶段：同名 concept 不覆盖、不静默合并，而是保留多个 active 节点。
- 扫描阶段：`POST /api/merge/scan` 产生 merge candidate。
- 执行阶段：`POST /api/merge/execute` 只执行已判断应合并的 decision，并归档旧节点。

## 4. 重点整合案例

### 4.1 候选合并：浅筋膜

涉及节点：

- `concepts/浅筋膜.md`
- `concepts/浅筋膜__doc_34ca6d21__ch_004__1.md`
- `concepts/浅筋膜__doc_34ca6d21__ch_005__1.md`

判断理由：

三者 `canonical_name` 均为“浅筋膜”，但来源章节不同：

- 绪论/基本操作中给出全身性定义：浅筋膜位于皮肤深面，含脂肪、浅血管、浅淋巴管和皮神经。
- 胸部章节强调胸部浅筋膜与颈部、腹部、上肢续行，并包含乳腺。
- 腹部章节强调腹前外侧壁浅筋膜较厚，并可分为 Camper 筋膜与 Scarpa 筋膜。

处理建议：

合并为一个“浅筋膜”综合节点，保留三条 `sources[]`，正文中分层写出通用定义、胸部特点、腹部特点。这样既去重，又不丢局部解剖差异。

### 4.2 保留：医学微生物学

`医学微生物学` 是学科级概念，关系指向 `微生物学`、`病原微生物`、`人体抗感染免疫`。当前没有同名候选，也不应因其抽象程度高而删除。它承担课程入口作用，适合作为微生物学图谱的上位节点保留。

### 4.3 保留：腹股沟管

`腹股沟管` 是局部解剖学中的结构节点，关系包含 `腹股沟韧带`、`腹股沟管深环`、`腹股沟管浅环`。它是腹前外侧壁教学链路中的关键结构，不应与一般“腹部”“腹腔”等上位区域节点合并。

### 4.4 保留并维持并列关系：消毒 / 灭菌

`消毒` 与 `灭菌` 在医学微生物学中相邻且相关，但定义不同：

- 消毒：杀死物体或环境中的病原微生物，不一定杀死芽胞。
- 灭菌：应消除所有微生物及其芽胞。

当前系统将二者以 `parallel` 关系连接，而不是合并。这符合教学需求：学生需要对比掌握二者边界。

### 4.5 保留：细胞膜

当前 `细胞膜` 来源于医学微生物学章节，定义语境是细菌细胞膜：位于细胞壁内侧，不含胆固醇，并承担转运、能量代谢和生物合成功能。虽然它可能与生理学中的“细胞膜”同名，但当前演示数据只有该来源，因此暂不生成跨教材 merge decision。若后续上传生理学教材，系统会保留同名多节点并创建 candidate，供人工或外部 scanner 判断是否合并。

## 5. 教学完整性说明

当前整合策略优先保护教学完整性：

1. **抽取期不静默合并**
   同名概念先保留为多个 active 节点，避免把不同章节、不同教材语境下的知识差异抹平。

2. **每个概念保留 source audit trail**
   frontmatter 中的 `sources[]` 保存 `textbook_id`、`chapter_id`、页码、定义和原文证据。合并后也不删除 source 记录。

3. **关系链路由 wikilink 派生**
   `## 关系` 中的 `[[wikilinks]]` 生成图谱边。合并执行时只重写 active vault 链接，不改 archive 和 decisions，从而兼顾展示一致性和审计原貌。

4. **不做自动删除**
   当前版本没有 remove 执行器。删除冗余节点风险高，可能破坏 prerequisite / contains 链路；因此 v1 只做 merge candidate 与 merge execute。

5. **局限**
   当前内置 scanner 只做 deterministic same-name，不做 embedding/LLM 同义识别。跨教材同义但不同名的概念，需要外部 GPT/Codex scanner 通过 `POST /api/merge/decisions` 写入 candidate。

## 6. 复现方式

1. 启动后端和前端：

```bash
./scripts/dev.sh
```

2. 上传教材 PDF/MD/TXT，等待章节解析与 LLM 抽取。

3. 在前端点击 `Scan merges`，或直接调用：

```bash
curl -X POST http://localhost:8000/api/merge/scan
curl "http://localhost:8000/api/merge/decisions?status=candidate"
```

4. 查看 graph 节点详情和 merge decision 详情。

5. 若要执行合并，调用 `POST /api/merge/execute`，提供人工或外部 agent 生成的 merged concept frontmatter/body。系统会写入 merged node，归档旧节点，并重写 active wikilink。
